## Maintenance

When adding a new genome:

**Single-file format (Track 2.0/3.0/4.1 — for shallow or function genomes):**
1. Create `{slug}.md` and `{slug}.meta.json` in this directory
2. Update this `INDEX.md` with the new entry
3. Declare `relations` in the new genome's frontmatter and meta.json
4. Sync to `template/.aioson/genomes/` so new projects also get it

**Modular folder format (Track 4.2 — for deep persona genomes):**
1. Create `{slug}/` folder with:
   - `SKILL.md` — lightweight manifest (~80-150 lines) with `references:` array in frontmatter
   - `manifest.json` — extended metadata
   - `references/` — subfolder with detail files; can have nested folders (e.g., `frameworks/`, `case-studies/`)
2. Update this `INDEX.md` with the new entry — use 📁 marker to indicate modular
3. Declare `relations` in `SKILL.md` frontmatter and `manifest.json`
4. Sync the entire folder to `template/.aioson/genomes/`

**Cognitive pipeline format (Track 4.3 — for deep persona genomes with full rigor):**
1. Track 4.2 folder structure PLUS additional references for the 9-layer pipeline:
   - `references/viability-assessment.md` (L1)
   - `references/behavioral-patterns.md` (L4a)
   - `references/cognitive-architecture.md` (L5)
   - `references/latticework.md` (L7)
   - `references/quality-report.md` (L9)
2. `manifest.json` extended with: `viability_score`, `pipeline_state`, `quality_report` (7 sub-scores), `sources` array (objects with id/authenticity/reliability), `ethics_mode`
3. Per-claim `source_ids` + `confidence` on top 20-30 most identity-defining claims (signature phrases, decision weights, axioms, values)
4. Update this `INDEX.md` with 🧬 marker to indicate cognitive pipeline
5. Optional: `implementation/system-prompt.md` generated on demand via `@genome export {slug} --as=system-prompt`
6. Sync the entire folder to `template/.aioson/genomes/`

When removing a genome:
1. Remove `.md` and `.meta.json` files (single-file) OR remove the entire `{slug}/` folder (modular)
2. Update this `INDEX.md`
3. Update `relations` in OTHER genomes that referenced the removed one
4. Sync removal to template

When upgrading single-file to Track 4.2 modular:
1. See `@genome` agent's "Migration from single-file to Track 4.2" section for the procedure
2. Decompose sections into appropriate references
3. Generate new `SKILL.md` and `manifest.json`
4. Remove the old single file (avoid ambiguity)
5. Update INDEX, sync template

## Format mix tolerance

Library is now uniformly modular. Agents resolve format automatically:
1. If `.aioson/genomes/{slug}/SKILL.md` exists → modular (Track 4.2)
2. Else if `.aioson/genomes/{slug}.md` exists → single-file (Track 2.0/3.0/4.1) — legacy compatibility, but no genomes use this anymore
3. Else → genome not installed
